# Word 採点チェッカー分類（MOS模擬アプリ正誤判定表251120_.csv ベース）

## ロジックが難しい（過程・手順の検証が必要）

**基準:** 結果だけでなく「ボタンを2回押す」などの手順が正しく行われたかの検証が必要なもの、または結果だけでは判定が困難なもの。

| プロジェクト-タスク | 問題文（要約） | 難しい理由 |
|---------------------|----------------|------------|
| **1-1** | 編集記号を非表示→再度表示 | **過程必須。**「非表示にしてから再度表示」のため、ShowAll（編集記号の表示/非表示）を**2回**実行したかが本質。結果だけ（最終的に表示ON）だと、1回しか押していない解答も正解になってしまう。 |
| **2-1** | 文字列を切り取って貼り付け | **過程必須。** 切り取り（Cut）と貼り付け（Paste）の手順が正しく行われたか、かつ元の位置に文字が残っていないかの検証が必要。結果だけではコピーとの区別が困難。 |
| **4-3** | 「エコと節約」のコメントを解決→削除 | 手順が複数（解決→解決済みへ移動→スレッドの削除→コメント一覧を閉じる）。「解決」と「削除」の両方を正しい順で行っているかの検証が必要。 |
| **4-7** | ドキュメント検査で透かし・ヘッダー/フッターのみ削除 | 「透かしとヘッダー・フッターだけ削除し、他は削除しない」という限定削除の検証。何が残り何が消えているかの切り分けが必要。 |
| **5-8** | 文末画像の背景を削除（保持する領域としてマーク） | **過程が本質。**「保持する領域としてマーク」で髪の毛・靴の部分をドラッグする手順が正しく行われたかの検証が難しい。結果の画像差分だけでは同一視しづらい。 |
| **7-1** | 互換モードを解除 | **過程の検証が必要。**「変換」から解除したかどうか。結果だけ（ファイル形式）だと別手段で変換した場合と区別しづらい。 |
| **7-4** | 「朗読会」でテキストファイルとして保存 | 別ファイル（.txt）の作成のため、ドキュメント本体の状態だけでは不十分。.txt の存在・内容の検証が必要。 |
| **7-5** | マクロ有効文書で保存し、読み取りパスワードを設定 | 別ファイル（.docm）の作成＋パスワード設定。ファイル存在・パスワードの検証は自動化が難しく、過程の検証も必要。 |
| **9-5** | 変更履歴を全て承認し、記録を停止 | 「全て承認」と「記録を停止」の両方が正しい手順で行われたかの検証。状態だけだと手順の違いが分かりにくい。 |
| **9-6** | 変更履歴のロック（パスワード「654」） | ロック状態とパスワードの設定を検証する必要がある。※CSVではパスワード「654」だが解答操作は「456」となっているため仕様要確認。 |
| **10-4** | 置換で「事例」を斜体に変更 | 「置換機能を使用して」が条件。手動で斜体にしただけでは不正解にしたい場合、置換を使ったかどうかの過程の検証が必要。 |

## 実装状況（今回の対応）

- **分類結果:** 上記のとおり。ロジックが難しい10件は既存の VSTO ログ利用または未実装のまま。
- **作成可能なもの:** 以下のチェッカーを結果ベースで実装・強化した。
  - **WordChecker1_3:** 3-2（セクション区切り `SectionStart == wdSectionNewPage`）、3-5（段区切り Chr(14) の検出）、3-6（1段目行間 1.3 倍数の検出）
  - **WordChecker1_5:** 5-2（画像の文字列の折り返し `wdWrapSquare`）、5-6（代替テキスト「セミナー案内」）
  - **WordChecker1_6:** 6-3（1つ目の表の1行目が4セル＝1行2列目が3列に分割）、6-4（表1行目の塗りつぶし/書式）
  - **WordChecker1_8:** 8-3（脚注番号書式 ①＝Ganada）、8-4（「参考文献一覧」先頭に「§」）
  - **WordChecker1_10:** 10-5（行間 1.6 をポイント 18～21 で判定）
- 5-3, 5-4, 5-5, 5-7, 5-8 および 10-2, 10-3 は Word OM の制約や手順検証のため、従来どおり簡易判定または未実装。
