## 1. PowerPointGrader のコンパイルエラー修正

**対象ファイル**: `PowerPointGrader.cs`

**内容**:
- `PictureFormat` のあいまいさ解消: `Microsoft.Office.Interop.PowerPoint.PictureFormat` を完全修飾で使用（変数宣言・代入時のキャスト）
- `TextFrame` のあいまいさ解消: `(Microsoft.Office.Interop.PowerPoint.TextFrame)sh.TextFrame` でキャスト、変数宣言も完全修飾
- `PpAction` が存在しない問題: 正しい列挙型は `PpActionType` のため、`PpAction.ppActionHyperlink` を `PpActionType.ppActionHyperlink` に変更
- `TextRange.Runs`: `Runs` はメソッド `Runs(int Start, int Length)` のため、`tr.Runs.Count` / `tr.Runs[r]` のループを、`tr.Runs(r, 1)` で 1 run ずつ取得する `while` ループに変更

## 6. Task 4 厳密判定（SlideID 追跡によるリアルタイム採点）

**対象ファイル**: `PowerPointGrader.cs`, `UiTestAppBarWindow.xaml.cs`

**内容**:
- **PowerPointGrader**  
  - 静的フィールド: `_previousSlideIds`（前回 Tick 時点のスライド ID 並び）, `Task4PassedByThirdSlideDeletion`（3枚目が削除されたと判定した場合 true）  
  - 静的メソッド `CheckSlideDeletion(List<int> currentSlideIds)`:  
    - `_previousSlideIds` が空なら比較せず現在の ID で更新して return  
    - 枚数が減っていなければ更新して return  
    - 減った場合: 消えた ID を `Except` で取得し、そのいずれかが `_previousSlideIds` のインデックス 2（3枚目）なら `Task4PassedByThirdSlideDeletion = true`  
    - 常に最後に `_previousSlideIds` を `currentSlideIds` で更新  
  - 静的メソッド `ResetTask4SlideDeletionState()`: アプリバー表示時用にリストとフラグをリセット  
  - `GradeProject1Task4()`: `Task4PassedByThirdSlideDeletion` が true のときのみ true を返す（従来の「非表示なし」判定は削除）
- **UiTestAppBarWindow**  
  - `_slideMonitorTimer`: 1秒間隔の `DispatcherTimer` を追加  
  - `InitializeSlideMonitor()`: `ResetTask4SlideDeletionState()` の実行後、タイマーを Start  
  - `SlideMonitor_Tick`: `Marshal.GetActiveObject("PowerPoint.Application")` → `ActivePresentation` → `Slides` を取得し、1～Count で `Slide.SlideID` を順に `currentSlideIds` に追加。`PowerPointGrader.CheckSlideDeletion(currentSlideIds)` を呼び出し。COM 解放を finally で実施  
  - `OnClosed` で `_slideMonitorTimer?.Stop()` を追加  
  - コンストラクタで `InitializeSlideMonitor()` を呼び出し

**結果**: 「何でもいいからスライドが減れば正解」ではなく、**3枚目（インデックス 2）のスライドが削除されたときだけ** Task 4 を正解とする厳密な判定になった。
